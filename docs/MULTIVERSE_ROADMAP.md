# 多周目语言宇宙路线图

## 核心规则

《代码觉醒者》的多周目不是平行宇宙，而是语言演化史。当前可游玩闭环只到 C 世界；后续世界通过“编译器纪元”逐步解锁。

演化顺序：

| 周目 | 语言 | 主题色 | 世界特点 | 核心机制变化 |
| --- | --- | --- | --- | --- |
| 1 | C | 霓虹蓝 + 绿 | 纯函数世界 | 指针、内存管理、结构体 |
| 2 | C++ | 霓虹紫 + 橙 | 面向对象城市 | 类、继承、多态、模板 |
| 3 | Java | 深蓝 + 金 | 企业虚拟机城邦 | 类加载器、接口、异常、垃圾回收 |
| 4 | Python | 白 + 金 | 极简主义花园 | 缩进、动态类型、包管理、装饰器 |
| 5 | Rust | 橙 + 铁灰 | 机械堡垒 | 所有权、借用、生命周期、unsafe |
| 6 | Assembly | 深红 + 黑 | 电子显微镜 | 寄存器、指令、栈、极端硬核 |

## 当前 C 世界收尾接口

运行时代码只保留最小扩展接口：

- `WorldRegistry.currentWorld` 固定从 `c` 开始。
- `WorldRegistry.switchWorld(worldKey)` 负责保存当前世界、调用退出函数、检查解锁条件、切换主题。
- `getThemeColor(slot)` 统一提供当前世界主题色。
- `getWorldStorageKey(worldId, key)` 为后续世界隔离 localStorage key。
- C 世界通关后调用 `onCWorldComplete()`，写入完成标记并显示“新的语言纪元等待开启”的日志。

## 编译器纪元过渡

语言世界之间通过“源代码之海”连接：

- 左侧流动已通关语言的代码。
- 右侧流动下一语言的源文件预告。
- 中央终端提示玩家输入 `next()`。
- 音乐从当前世界主题逐层过渡到下一世界主题。

## 后续世界概念预览

### C++ 世界

- 类 = 建筑蓝图。
- 继承 = 楼层叠加。
- 多态 = 变形门。
- 模板 = 可配置陷阱。
- Boss = 菱形继承恶魔。

### Python 世界

- 缩进 = 悬崖。
- 列表推导 = 工厂流水线。
- 动态类型 = 变色龙 NPC。
- 装饰器 = 函数光环。

### Rust 世界

- 所有权 = 唯一钥匙。
- 借用 = 限时通行证。
- 生命周期 = 能量条。
- unsafe 区域 = 禁区。

## 扩展公约

新增语言世界必须遵守：

1. 实现 `initXxxWorld()` 和 `closeXxxWorld()`。
2. 在 `WorldRegistry` 注册世界包。
3. 存档使用 `worldId` 前缀。
4. 所有视觉色彩通过 `getThemeColor()` 获取。
5. 当前世界的教程内容不得污染其它语言世界。
