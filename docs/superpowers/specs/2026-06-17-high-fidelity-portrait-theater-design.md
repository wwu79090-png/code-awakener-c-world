# 高精度动态立绘剧场设计

## 背景

《代码觉醒者》当前是单文件 Phaser/HTML C 语言学习 RPG。新视觉目标不是再加一层网页装饰，而是把新玩家第一次进入游戏的开场改造成“高精度 2D 动态立绘剧场 + 赛博朋克科技交互”体验。用户已确认三项关键方向：

- 交付形态：直接替换主游戏首次流程，而不是独立演示页。
- 触发范围：新玩家首次进入游戏时直接进入“变量学舱剧场”，现有概述导师开场弱化或后置。
- 角色审美：轻量 Live2D 风格，干净、可拆件、适合稳定动画，不走厚涂写实。

## 目标

1. 新玩家第一次进入游戏时看到变量章节剧场：学舱门暗转蓝、扫描线、导师全息显影、毛玻璃对话 UI、关键词高亮、代码光带加速、黄金箭头进入编辑器。
2. 主角、导师、商人和教学 NPC 建立统一高分辨率 WebP atlas + 拆件元数据协议，支持后续替换服装和扩展表情。
3. 对话 UI 从静态文本框升级为底部 35% 剧场面板，包含逐字文本、关键词金色回弹、控制台输入框、光带进度条。
4. 学习场景呈现对称冷蓝赛博走廊，学习舱门显示三种进度光态。
5. 桌面端追求高沉浸视觉；移动端最低目标 60fps，通过降特效保帧，不允许主动锁到 30fps。

## 非目标

- 不引入真实 Live2D 或 Spine runtime。实现采用 WebP atlas、拆件元数据、DOM/CSS transform 和少量 Canvas/Phaser 动效模拟。
- 不重写整套课程系统、存档格式或编辑器编译逻辑。
- 不把全部游戏 UI 一次性改成新剧场风格。本次只替换首次开场和变量章节入口相关视觉链路，并建立角色资源协议。

## 架构选择

采用混合剧场：

- Phaser 负责主世界镜头、学习走廊、舱门状态、地面光轨、扫描线、黄金箭头和环境粒子。
- DOM 负责 1200x1600 立绘层、底部毛玻璃对话面板、逐字文本、关键词动画、控制台输入框、进度光带和移动端重排。
- JS 控制器协调两层：`VariableTheaterController` 管理流程状态，调用 Phaser 场景效果并驱动 DOM 立绘/对话状态。

理由：Phaser 适合世界空间和粒子，DOM 更适合清晰文字、毛玻璃和高 DPI 立绘布局。这个拆分能保持现有单文件游戏稳定，也能避免在 Phaser Text 中硬做复杂富文本动画。

## 首次流程

1. 新玩家进入游戏世界后，检测 `gameState.progress.openingNarrativeSeen === false`。
2. 不播放现有概述开场终端全文，改为设置 `openingNarrativeSeen = true` 并进入变量剧场。
3. 摄像机移动到变量学舱视角，玩家输入冻结，背景音进入低频环境层。
4. 变量舱门从暗灰转为蓝色呼吸光。
5. 屏幕播放一次水平扫描线，边缘出现 3px 轻微红/青色差。
6. 导师 NPC 右侧全息显影，代码光带独立旋转。
7. 主角修复师左侧进入，身体向内倾斜，呼吸幅度为 0.5px。
8. 底部 35% 毛玻璃面板弹起，导师名头衔单独一行显示为霓虹蓝。
9. 对话逐字显示：`变量不是数字本身，而是给数据命名的舱位。先写 int，再让 printf 说出它。`
10. “变量”、“int”、“printf” 在语音到达时 0.1 秒内变为霓虹金色并放大回弹。
11. 关键词触发导师代码环加速旋转一圈。
12. 对话结束后地面黄金箭头延伸到编辑器入口，恢复输入并打开或提示打开变量编辑器。
13. 现有概述课程仍保留在课程列表中，后续作为可回放学习节点，不阻断首次剧场。

## 角色资源协议

所有角色视觉资源输出为 WebP。每个角色拥有 atlas 图和 JSON-like 元数据，元数据直接内嵌到单 HTML，避免额外加载失败。

主角修复师：

- atlas 逻辑画布：1200x1600。
- 拆件：头、前发、后发、左眼、右眼、嘴、躯干、左上臂、左前臂、右上臂、右前臂、左手、右手、独立手指组、服装层、发光边缘层。
- 动作：0.5px 呼吸、视线 XY 跟随、编译成功眼睛金色高光、失败低头嘴角下压、代码崩溃时 0.2 秒横向像素错位。
- 换装接口：`outfitKey` 指向 atlas 内服装层矩形，默认 `repairer-default`。

导师 NPC：

- atlas 逻辑画布：1200x1600。
- 分层：身体实体层、胡须层、瞳孔层、眉毛层、手臂/手指层、代码光带层、全息噪声层。
- 表情：稳重微笑、轻抚长须、微皱眉、瞳孔收缩。
- 代码光带独立渲染，关键词到达和里程碑时加速旋转。

商人 NPC：

- atlas 逻辑画布：1200x1600。
- 半透明渐变衣物层、交易前推动画层、手部展示层、商品光效层。
- 首次变量剧场不强制出场，但资源协议和基础 atlas 必须存在。

教学 NPC：

- 统一灰色工装模板。
- 通过头部层和护目镜颜色层区分角色。
- 首次变量剧场不强制出场，但模板 atlas 必须存在。

## 对话 UI

- 位置：桌面端固定在屏幕底部 35% 高度。
- 背景：60% 不透明度磨砂玻璃；桌面端使用 `backdrop-filter: blur(8px)` 透出背景地图。
- 字体：JetBrains Mono 等宽字体，中文 fallback 使用现有系统字体。
- 角色名：独立一行，霓虹蓝，内容为 `老程序员 · 变量引路人`。
- 正文：逐字显示，关键词按 token 触发高亮。
- 输入框：控制台 `>` 前缀，竖线光标 0.5 秒呼吸闪烁。
- 进度条：细光带样式，更新时使用过冲回弹。
- 移动端：宽度小于 800px 时改纵向排列；主角头像缩至左下，导师头像缩至右上；毛玻璃降级为半透明纯色。

## 场景与后处理

学习走廊：

- 对称构图，深蓝冷灰基调。
- 地面单向流动蓝色光轨，里程碑达成时短暂加速。
- 墙壁科技凹槽，低密度半透明白色空气粒子。
- 学习舱门上方全息标牌使用 Orbitron 风格：`#01 变量`。
- 舱门状态：蓝色呼吸光为当前活跃，暖金色稳态光为已通关，暗灰为未解锁。

后处理：

- 所有霓虹元素带泛光。
- 加载场景或编译失败时启用屏幕边缘 3px 红/青色差。
- 崩溃故障时主角拆件和 UI 边缘做 0.2 秒横向像素错位。

## 性能策略

桌面端：

- 默认启用毛玻璃、泛光、色差、故障错位、环境粒子和代码光带。
- 使用 `requestAnimationFrame` 统一驱动 DOM 剧场动画，避免多组独立定时器。

移动端：

- 最低目标 60fps，不主动锁 30fps。
- 宽度小于 800px 时禁用色差与故障错位，粒子数量减至 50%，毛玻璃改半透明纯色。
- 若运行时帧时间连续 20 帧高于 16.7ms，则继续降低粒子、暂停非关键代码雨、减少阴影/滤镜，但仍以恢复 60fps 为目标。
- 所有点击目标保持现有移动端触摸可用性，不覆盖虚拟摇杆、菜单抽屉和编辑器安全区逻辑。

## 与现有系统的衔接

- 替换 `showOpeningNarrativeTerminal(startMentorDialogue)` 所在的新玩家开场链路。
- 保留 `OPENING_MENTOR_LINES` 作为后置可回放内容，不删除课程解释。
- 首次剧场结束后继续沿用现有任务、课程、编辑器和存档逻辑。
- 新增的 `variablesTheaterSeen` 标记写入 `gameState.progress`，防止老玩家和已看过玩家重复强制触发。
- 对旧存档：若 `openingNarrativeSeen` 已为 true，不自动强制播放；可通过调试或菜单回放入口触发。

## 测试与验收

自动测试需要覆盖：

- 新资源协议常量存在，包含主角、导师、商人、教学 NPC 的 WebP atlas 条目和拆件元数据。
- 新玩家开场链路调用变量剧场，不再直接调用旧概述开场。
- 剧场结束后设置 `openingNarrativeSeen` 和 `variablesTheaterSeen`，并恢复玩家输入。
- 对话 UI 包含底部 35%、毛玻璃、JetBrains Mono、关键词 token、控制台输入和光带进度条选择器。
- 移动端 CSS 包含小于 800px 的纵向布局和 60fps 保帧降级策略。
- CSP style/script hash 更新后仍通过。
- `npm test` 通过。
- `npm run build` 通过。
- 浏览器验证桌面与移动视口，确认首屏剧场非空、角色不遮挡文本、关键词动画可见、流程能进入编辑器。

人工验收标准：

- 左右立绘不再是几何占位，必须达到轻量 Live2D 视觉小说级角色质量。
- 主角和导师边缘在高 DPI 下清晰，不出现明显锯齿。
- 对话面板占底部 35% 且文字可读。
- 移动端不得因特效导致明显卡顿；实现应以 60fps 为最低目标做降级。
- 整体观感脱离网页小游戏廉价感，首屏像一个完整游戏剧场，而不是网页弹窗。
