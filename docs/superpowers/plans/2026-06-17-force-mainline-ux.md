# Code Awakener v1.0.25 强制主线化 UX 计划

## 目标
- 菜单只保留能帮助推进当前关卡的核心入口，移除会误导玩家手动进化、手动调难度、手动换外观和重复教学的入口。
- 世界进度只由收集记忆碎片、完成编译和主线事件自动推进，不再由空格键或菜单按钮触发。
- 每个记忆碎片有图像化表现、明确位置提示和未收集追踪，降低卡关率。
- NPC 交互先出现聚合选择，再进入交流、任务、商店或线索。
- 休息提示、公告提示、缩小按钮和音频解锁按钮不再打断体验。

## 实施步骤
1. 更新测试：删除旧的菜单/空格进化/手动星盘断言，加入新流程断言。
2. 精简菜单：学习辅助仅保留手动保存和游戏设置；移除星盘、世界之镜、学习报告、家长说明、练习回放、难度、外观和重置进化入口。
3. 改造流程：角色创建完成后弹出难度和外观选择，保存后进入游戏。
4. 改造推进：空格只用于跳过/继续文本；碎片和编译成功自动推进世界进度，星盘升级自动智能分配。
5. 改造引导：碎片列表显示具体区域、坐标、方向和状态；隐藏碎片给出精准提示。
6. 改造 NPC：靠近 NPC 后先显示交流/任务/商店/线索选择。
7. 改造音乐：加入分段主题、场景层、碎片/编译触发变奏和长时间不重复的旋律队列，避免背景音乐反复单调循环。
8. 改造公告：启动公告只显示当前版本内容并延长停留时间，历史更新内容移到主菜单独立入口。
9. 打磨提示：休息提示间隔拉长，右下角音频按钮仅在浏览器需要手势解锁时出现，通用缩小按钮对齐灵动岛左上角关闭按钮基线。
10. 验证：运行单元测试和构建/静态检查，记录仍无法覆盖的浏览器项。
