# 《代码觉醒者》性能与稳定性优化说明

日期：2026-06-15

## 本次实现范围

- 新增三档性能模式：高画质 60fps、中画质 45fps、低画质 30fps。
- 新增启动 5 秒性能基准测试：根据平均 FPS、CPU 核心数、设备内存自动选择运行档位。
- 新增 F1 性能仪表盘：显示 FPS、帧耗时、CPU/GPU 估算、Draw Call、粒子数量、内存、对象池状态。
- 新增性能 Governor：低于目标帧率持续 2 秒记录性能日志；连续 5 秒低于 30fps 强制切低画质；低于 15fps 快速保护。
- 新增对象池：
  - 粒子池上限 100。
  - UI 临时卡片池上限 20。
  - 特效池上限 10。
- 新增空闲 GC/资源修剪：玩家 30 秒无输入且距离上次修剪超过 10 分钟时，清理对象池与音频缓存。
- 新增错误日志与友好弹窗：全局异常会写入 localStorage，并可导出为 `error_log.txt`。
- 新增编辑器新手引导：完整 / 精简 / 无三种模式，首次打开自动显示，可在设置中重置。
- 强化关卡完成特效：严格按“裂缝代码涌现 -> 字符流动 -> 路径固化 -> 全屏脉冲 -> 终端宣告”顺序播放。

## 2026-06-15 追加修复

- 解除项目内 FPS 限制：
  - 移除设置界面的“帧率上限”控件。
  - Phaser 启动配置不再设置 `fps.target`。
  - Arcade Physics 改为非固定步进，避免内部固定 60fps。
  - 性能模式中的帧率数值仅作为“低帧降级阈值”，不再作为上限。
- 移除“电影级 / Cinematic”画质选项：
  - 设置界面不再暴露 `cinematicQualitySelect`。
  - 渲染管线不再自动进入 `cinematic` 档，旧存档里的 `cinematic` 值会兼容降级到 `high`。
  - 保留高/中/低稳定画质与 LUT、SSR、雾、草地等独立开关。
- 修复创角完成后立即报错/崩溃：
  - 新增 `codeGenesisCompletionInFlight` 防重入锁。
  - `return 0;` 触发交接后忽略重复输入，避免重复保存/重复加载。
  - 创角状态丢失或存档验证失败时进入友好错误路径，而不是抛出未处理异常。

## 统一防崩溃系统

- 新增 `SafeExecute.run()` / `SafeExecute.runAsync()`，用于包裹关键同步与异步操作。
- 新增 `SafeBehaviour` 浏览器等价基类，负责定时器、事件监听和销毁清理。
- 新增 `SafeLoadScene()`，场景切换前校验目标场景白名单，失败时回主菜单并记录日志。
- 新增 `ErrorLogManager`，错误日志保存在 localStorage，启动时自动清理 3 天前日志。
- 错误弹窗只显示友好文案；普通错误只提供“我知道了”，严重错误才显示“重新开始”。
- 日志按钮会尝试复制日志到剪贴板，同时下载 `error_log.txt`。

## 浏览器版替代说明

用户需求里包含 Unity 专属词汇，例如 VFX Graph、Compute Shader、ASTC/BC7、`Application.persistentDataPath`。当前项目是单 HTML 浏览器游戏，因此本次采用等价实现：

- VFX Graph / Compute Shader：用 Phaser 对象池、粒子 LOD、帧间隔调度替代。
- ASTC/BC7：浏览器单文件无法强制平台纹理压缩，采用减少粒子与代码雨更新、图集/程序贴图复用来降低压力。
- `error_log.txt`：浏览器不能静默写本地文件，改为 localStorage 持久化，并提供导出按钮下载 `error_log.txt`。
- ThreadPool/Job System：使用 Web Worker 处理导师实时解析，Worker 不可用时回退主线程轻量解析。

## 关键降级策略

- 高画质：粒子最多 300，代码雨每 4 帧更新，导师实时解释完整开启。
- 中画质：粒子减半，代码雨每 8 帧更新，导师解释降级为错误优先。
- 低画质：关闭代码雨和非核心粒子，关闭实时解释，只保留核心玩法。

## 验证重点

- 进入游戏后按 F1 可打开性能仪表盘。
- 设置 -> 画面中可切换性能模式、帧率上限、编辑器引导模式。
- 打开编辑器时首次出现右侧引导层，按下对应快捷键会高亮卡片。
- 完成关卡后，代码铺路动画按顺序播放。
- 执行 `triggerNullReferenceTest()` 后会弹出友好错误窗口，并可导出错误日志。
